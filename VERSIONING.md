# Versioning and editions

The handbook separates editorial stability from continuous improvement.

- **`main`** is the living edition. It may receive corrected links, new workflows, translations and small clarifications after review.
- **Major releases (`v1.0`, `v2.0`)** are formally reviewed editions with a frozen release manifest and archived snapshot.
- **Minor releases (`v1.1`, `v1.2`)** add substantial reviewed content without changing the overall intellectual architecture.
- **Patch releases (`v1.0.1`)** correct errors, broken instructions or metadata without changing the teaching argument.

Every formal release should include:

1. a Git tag and GitHub release;
2. a release manifest listing all included pages and translation status;
3. a changelog;
4. a citable archive with a persistent identifier, when available;
5. an optional PDF/EPUB snapshot for review, deposit and offline access;
6. the names and scope of the reviewers for the stable handbook core.

The canonical scholarly reference is always a numbered release, not an undated view of `main`.
