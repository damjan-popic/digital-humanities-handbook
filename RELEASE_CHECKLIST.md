# Release checklist

## Development release

- [ ] `make check` passes on a clean environment.
- [ ] Translation coverage report is committed.
- [ ] Changelog and `CITATION.cff` show the same version/date.
- [ ] New third-party material has rights metadata.
- [ ] Site language switch works on home, chapter and workflow pages.
- [ ] A source archive and built-site archive are created with checksums.

## Formally reviewed major release

- [ ] Stable-core content freeze declared.
- [ ] Scholarly and didactic reviewers received the exact release candidate.
- [ ] Required changes resolved and final approval recorded.
- [ ] Both stable language versions received language/terminology review.
- [ ] Accessibility and link audit completed.
- [ ] Publisher agreement confirms open licence and living-edition continuation.
- [ ] Persistent identifier and institutional deposit arranged.
- [ ] Version tag points to the reviewed commit.
- [ ] Review statement and preferred citation published.
- [ ] Development resumes only after the archived release is immutable.
